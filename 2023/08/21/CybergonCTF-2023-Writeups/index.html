CybergonCTF 2023 Writeups

Narongpol Kijrangsan
2023-08-21

Introduction

On 19th August to 21st August I participated in CybergonCTF 2023 which is a CTF that prepare especially for Myanmar players and only Myanmar citizens are eligible to the rewards. However I as a Thai person still decided to participate in this CTF as just for the sake of fun and learning, and the luckicy it is really fun and is a good learning opportunities, I also encountered many type of problems that I have never seen before which give me many new knowledges in forensics, steganography, cryptography, binary exploitation, and many more! People in the CTF Discord also friendly too, as they very happy to share writeups to other players who want to know how to solve. And here is my writeups, and I hope that whoever read this can gain new knowledge and insights!

Rank: 22nd
Score: 2290 points

teeny

Score: 235
Attachment: teeny

This can be solved by using ropper or ROPGadget to find ROP gadgets and we can inject ROP chain by padding our input 8 bytes until hit the return pointer and then we place our ROP chian

autograph

Score: 395
Attachment: autograph

The idea of this challenge is to leak elf base address and libc base address using format string vulnerability and overwrite GOT table to replace printf function with system function, so the next time we call it, it will invoke system function

Flag: CybergonCTF{PHorM47_57rin9_I5_d4N93rou5}

really random

Score: 350
Attachment: random

The challenge will ask a user to guess number, luckily the user can overwrite the random seed which make it predictable, in this case we overwrite it with 0 and store the answer.

After we manage to guess all the numbers correctly, program will execute ret instruction which allow us to use ROP chain, which is impossible in case we guess the numbers incorrectly because it will call exit function and have no return

Flag: CybergonCTF{noT_RE4LlY_r4NdOM?}

Cryptography

EZ RSA

Score: 100
Attachments: Crypto3.py output.txt

Understanding the basic principles of RSA and brute forcing p and q value that cause p * q = n is enough to solve this challenge. We try to XOR given p and q with num in range [512, 512+16) because we know that the bit that cause the XOR result to have 512 value bit have to be from num and not p and q because p and q can only be in range [0, 9] due to the digits splitting in the challenge script
